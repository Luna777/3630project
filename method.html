Method

Sample code:
public class MyProgram{
    public static void main(String[] args){
        int numb1 = 10;
        int numb2 = 5;
        int sum = sumNumbers(numb1, numb2);

        System.out.println( numb1 );
        System.out.println( numb2 );
        System.out.println( sum );
    }
    public static int sumNumbers(int n1, int n2){
        int n3 = n1 + n2;
        return n3;
    }
}
Output:
10
5
15

In the last topic, we sum the two integers by using "+" in the statement. Now let's try another way. Let's call a function to make a sum.
In the sample code, there is another method that looks like main method but under it. It is a part of our class MyProgram. It called sumNumbers:
    public static int sumNumbers(int n1, int n2){
        int n3 = n1 + n2;
        return n3;
    }

The title line of the method has several elements. "public" means the method is open to public that other statement could call it. "static" means it is a static method that always loaded in memory. "int" is the return type and it means calling the method could get an integer result back. "sumNumbers" is the name of method, and it followed by parentheses. Inside parentheses, "int n1" and "int n2" are the input parameters, and they are separated by comma.
The content are two statements. "int n3 = n1+ n2;" is making n3 a sum of input parameters n1 and n2. "return n3; " is going to return the value of n3. We just said that this method's return type is "int", that means "n3" which is the value that returns must be an "int", otherwise it will be a mistake.
Overall, method "sumNumbers" is going to read in two integer input parameters, and make a sum of them, then return the value of the sum.

Now, let's use the "sumNumbers" method. In the main method, we have two integers "numb1" and "numb2", and they are 10 and 5. The 3rd int is "sum". The value of "sum" is the return value of method "sumNumbers(numb1, numb2)" which is the sum of "numb1" and "numb2", so "sum" is going to be 15.
The output is showing the values of numb1, numb2 and sum.
