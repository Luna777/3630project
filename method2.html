Method

Sample code:
public class MyProgram{
    public static void main(String[] args){
        String drink = kitchen("apple and water");
        System.outprintln( drink );

        String food = kitchen("water and raw rice");
        System.out.println( food );
    }
    public static String kitchen(String items){
        if(items == "water and raw rice"){
             return "cooked rice";
        }
        else if(items == "apple and water"){
             return "apple juice";
        }
        else{
             return "empty";
        }
    }
}
Output:
apple juice
cooked rice

This is a more practical example of using method. In the sample code, method "kitchen" is going to read a String parameter items. The return type of "kitchen" is also String.
Inside the method, there are a set of if-else statement. For the details of using if-else statement, we are going to explain them later (maybe in next app). Here is a simple explanation:
        The first if is asking does "items" equal to "water and raw rice", if true, return the String "cooked rice", if false, go to the second condition. The second condition (else if) is asking does items equal to "apple and water", if ture, return "apple juice", otherwise go the third condition (else). The last "else" is for every other situations, and it will return "empty".

In the main method, we set up a String "drink" and we let the method "kitchen" read in "apple and water", then the return value of kitchen, "apple juice", is assigned to "drink".
The next example "String food = kitchen("water and raw rice");" is the same.

Therefore, by using method "kitchen", we get our variables "drink" and "food" done.

From the method "sum" and method "kitchen" examples, we learned that method is doing more actions for our program. It is made of some related statements that will doing a particular job, such as doing sum or as kitchen job. It is like a functional room in a house, like a room of doing calculation or a room as kitchen to bring in items and bring out well made food. A method could simple or complicated as we need.
